# Clarify Before Execute

## Use When

- 任务存在多种理解路径
- 一旦走偏就会返工
- 输出需要交接、留档或持续迭代

## Why It Helps

复杂任务里最昂贵的失败通常发生在执行前，而不是执行中。先对齐目标、边界、交付物和验收标准，往往比后面多做很多补救更省。

## Recommended Move

1. 先问一个最影响方向的问题
2. 做一次完整共识确认
3. 再写当前任务的一次性执行指令
4. 最后才进入执行

## Common Misuse

- 对非常简单的任务也强行套完整流程
- 用很多问题堆出“专业感”，但没有真正推进理解
