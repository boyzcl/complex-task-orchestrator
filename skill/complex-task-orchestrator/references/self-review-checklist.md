# Self-Review Checklist

Use this reference after executing the task.

## Required Review Flow

1. Score the result from 1 to 10.
2. Explain every deduction.
3. Revise the work if the score is below 9.5.
4. Repeat until the score is above 9.5, or state the concrete external blocker that prevents further improvement.

## Review Questions

- Did the work solve the real problem rather than a shallow proxy?
- Did the execution stay aligned with the confirmed requirement consensus?
- Did the deliverables actually get created and validated?
- Did the work leave enough artifacts for future reuse or handoff?
- Were assumptions disclosed when full clarification was not possible?
- Is the final result strong enough for the intended user, not just technically complete?

## Required Final Summary

Always end with:

- what the problem was
- what the root cause was
- what solution was implemented
- what confidence percentage you assign to the current solution
