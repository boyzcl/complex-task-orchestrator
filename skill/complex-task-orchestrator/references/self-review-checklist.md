# Self-Review Checklist

Use this reference after executing the task.

## Required Review Flow

1. Run the preset validation/test plan before self-review.
2. Record pass, fail, blocked, or skipped status for each required validation point.
3. If a must-pass validation point fails, revise the work and rerun the relevant validation before scoring.
4. Score the result from 1 to 10 only after the verification pass.
5. Explain every deduction with reference to validation results where possible.
6. Revise the work if the score is below 9.5.
7. Repeat until the score is above 9.5, or state the concrete external blocker that prevents further improvement.
8. State clearly whether the main task is complete or whether the remaining work is only forward testing / extended validation.

Self-review cannot replace preset validation. 自检不能替代预设验证。If required validation fails, do not assign a high score just because the answer looks polished.

## Review Questions

- Did the work solve the real problem rather than a shallow proxy?
- Did the execution stay aligned with the confirmed requirement consensus?
- Did the validation/test plan exist before the execution plan?
- Did the deliverables actually get created and validated?
- Did all must-pass validation points pass, or are blockers explicit?
- Did the work leave enough artifacts for future reuse or handoff?
- Were assumptions disclosed when full clarification was not possible?
- Is the final result strong enough for the intended user, not just technically complete?

## Required Final Summary

Always end with:

- what the problem was
- what the root cause was
- what solution was implemented
- whether preset validation passed
- what confidence percentage you assign to the current solution
- whether the main task is complete and whether any remaining work is optional extended validation
