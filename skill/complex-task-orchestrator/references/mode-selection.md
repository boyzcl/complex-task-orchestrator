# Mode Selection

用本参考决定当前任务应该走轻量模式还是完整模式。

## 轻量模式

适用条件：

- 任务目标已经清楚
- 关键交付物清楚
- 关键约束清楚
- 误解成本不高
- 用户明确希望直接推进

轻量模式下：

- 可以减少追问轮次
- 仍要做简短共识确认
- 仍要列出最小验收检查
- 仍建议生成执行计划文档，但可以把验证方案和执行计划合并成短版
- 完成后仍要按最小验收检查回验

## 完整模式

出现以下任一情况，优先进入完整模式：

- 任务开放性很强
- 任务有多阶段交付
- 用户需求明显不完整或容易被多种方式理解
- 错误代价高
- 需要严格方法论
- 用户明确要求“一次一个问题”“先问清楚再做”
- 你判断如果不深入澄清，后续极可能返工

即使当前场景不适合多轮追问，只要上述条件仍成立，也不应自动降为轻量模式。此时应保留完整模式的框架，只是把未确认部分改为显式假设继续。

完整模式下：

- 使用单问澄清
- 明确做需求复述和确认
- 先生成测试与验收方案
- 再生成完整的一次性执行计划文档
- 执行完成后按预设验证回验，失败时迭代
- 严格区分主任务执行与扩展验证

## 快速判断清单

只要以下 6 项中有 2 项及以上回答为“否”，就不要用轻量模式：

1. 目标是否已经清楚
2. 交付物是否已经清楚
3. 约束是否已经清楚
4. 验收标准是否已经清楚
5. 最小验证方式是否已经清楚
6. 如果我现在直接执行，是否大概率不会走偏
