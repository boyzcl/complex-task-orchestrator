# Validation Plan Template

用本模板为当前任务生成测试与验收方案。

## 基础模板

```md
# [任务标题] 测试与验收方案

## 验证目标

[说明本次验证要证明什么，哪些结果必须成立。]

## 验证范围

- 必须验证：
- 可以抽查：
- 本轮不验证：

## 验证用例

| ID | 验证点 | 方法 | 预期结果 | 证据 | 是否必过 |
| --- | --- | --- | --- | --- | --- |
| V1 | [核心需求是否覆盖] | [对照需求共识检查] | [所有必做项都有交付] | [文件/命令/截图/表格] | 是 |

## 失败迭代规则

- 必过项失败时，不进入最终总结。
- 先定位原因，再修复。
- 修复后重新运行相关验证。
- 若无法修复，说明外部阻塞、影响范围和剩余风险。
```

## 任务类型映射

### 代码任务

常用验证：

- 单元测试
- 集成测试
- 构建
- lint
- 类型检查
- 手工 smoke test

### 前端任务

常用验证：

- 浏览器截图
- 控制台错误检查
- 交互检查
- 响应式布局检查
- 关键视觉状态检查

### 文档任务

常用验证：

- 结构完整性检查
- 术语和口径一致性检查
- 证据支持检查
- 读者路径检查
- 链接和文件引用检查

### 研究或证据整合任务

常用验证：

- 证据等级检查
- 关键结论引用覆盖
- 反证和不确定性边界检查
- 相关性与因果表述检查
- 来源时效性检查

### Skill / Workflow 任务

常用验证：

- 触发边界检查
- Quick Path / Full Path 选择检查
- 默认产物检查
- 失败路径检查
- runtime reuse 边界检查
- 安装或结构校验

### 项目整改任务

常用验证：

- 文件存在检查
- 自动化脚本通过
- README 与实际状态一致
- validation matrix 更新
- changelog 更新
