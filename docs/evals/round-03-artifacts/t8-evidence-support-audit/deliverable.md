# T8 Evidence Support Audit

日期：2026-04-17

范围：仅检查以下 5 个固定来源中的“表述偏满”或“证据链不够紧”之处，不展开新评估。

## 发现

1. `docs/evals/product-value-decision-2026-04-17.md:21-22` 把 `T3` 和 `T4` 写成“已经能稳定省事”，表述偏满。  
证据问题：当前固定来源里，`T3` 与 `T4` 各只有 1 个完成样本；`docs/evals/round-03-task-board.md:19-22` 只支持“这两类任务在本轮样本中表现好”，还不够支持“稳定”。同时同一决策文档在 `55-61` 也明确要求继续扩样本。  
更稳的替代表述方向：改成“`T3` 和 `T4` 提供了初步证据，显示在短任务和快速判断任务里有较高概率省事，值得进入受限试用并继续扩样本验证稳定性”。  

2. `docs/evals/round-03-assessment.md:55-61` 中“`T1` 在聚焦复跑后也证明了减少手动补充上下文”这一句，证据链不够紧。  
证据问题：`docs/evals/round-03-task-board.md:19` 只说明 `T1` “聚焦复跑后产出可用 memo，证明复杂决策任务有价值但需要更干净的输入边界”；`docs/evals/round-03-assessment.md:19-22` 也显示它首轮 `timeout`、复跑才完成。现有来源更能支持“收紧边界后可产出结果”，但还不足以直接支持“明确减少了手动补充上下文”。  
更稳的替代表述方向：改成“`T1` 更像是在说明：收紧输入边界后可以产出可用结果；至于是否减少了手动补充上下文，当前证据仍偏弱”。  

3. `docs/evals/round-02-assessment.md:119-121` 中“在 `Codex` 客户端内稳定成立”表述偏满。  
证据问题：同文 `28-31`、`37-40`、`95-101` 以及 `docs/evals/client-trigger-log.md:28-31,38-41,54-55` 显示，当前 `Codex` 侧只有 `C1`、`C2` 两个任务位；其中 `C1` 需要聚焦复跑，`C2` 还存在客户端收尾卡住、靠 runner 隔离获得完整结果的情况。现有证据足以支持“客户端内已出现可复现正例”，但还不够稳到“稳定成立”。  
更稳的替代表述方向：改成“项目已经从‘只在 repo 内成立’推进到‘在 `Codex` 客户端内获得初步且可复现的成立证据’，但稳定性仍需更多不依赖复跑/隔离的样本支撑”。
