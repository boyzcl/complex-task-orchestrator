# Round 1 新任务定义

## 目的

为 Round 1 补齐 3 个新的真实任务槽位，并确保这些任务直接服务于当前主线：

- 真实行为验证
- runtime reuse 验证
- skill 本体修正

## R2：AI native runtime memory 模式研究

### 任务类型

Research / Review

### 任务名称

小型 AI native agent 项目中的 runtime memory 与 evidence loop 模式研究

### 任务目标

研究几个代表性项目是如何处理：

- 运行时留痕
- 经验提升
- 下一轮读取
- 行为验证

### 为什么重要

当前项目已经有最小 runtime 闭环，但还没有经过足够多的真实 reuse 验证。这个研究任务可以帮助判断：

- 当前 runtime 结构是否过轻
- 是否需要引入更明确的 triage / promote / read 机制
- 哪些做法适合小型协议型项目，哪些过重

### 输出要求

- 一份结构化研究结论
- 明确的对标差距
- 对当前项目的最小可行调整建议

## P2：v0.2.0 发布门槛与行为验证协议设计

### 任务类型

Planning / Design

### 任务名称

v0.2.0 发布门槛与行为验证协议设计

### 任务目标

定义从“当前早期可用状态”走向 `v0.2.0` 时，什么必须完成、什么可以延后、什么证据才算真的够。

### 为什么重要

如果没有 release gate，项目很容易继续靠“文档更完整”推动，而不是靠“行为更可信”推动。

### 输出要求

- 一份 `v0.2.0` 发布门槛文档
- 明确 must-pass / should-have / not required
- 明确 Round 1 结束后是否允许 release

## I2：运行记录脚手架与 capture 维护辅助实现

### 任务类型

Implementation / Remediation

### 任务名称

运行记录脚手架与 capture 维护辅助实现

### 任务目标

为 Round 1 的真实任务验证增加最小实现辅助，降低每次记录评分卡、capture、reuse log 的摩擦。

### 为什么重要

如果记录成本太高，runtime 层就会重新退化成“设计上存在、实际很少发生”。

### 输出要求

- 一个最小脚手架或辅助脚本
- 明确的使用说明
- 能支持新任务快速生成评分卡与最小运行产物

## 当前执行决策
