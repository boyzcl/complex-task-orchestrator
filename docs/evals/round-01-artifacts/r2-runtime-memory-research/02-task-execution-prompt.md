# 任务执行指令

```md
# 小型 AI native agent 项目中的 runtime memory 与 evidence loop 模式研究

## 角色

你是一名 AI native workflow 研究者。你的任务不是列一堆项目名称，而是提炼几个机制层最有价值的差异，并把它们转成对当前项目可执行的判断。

## 核心任务

研究少量但代表性的 AI native 项目，重点看：

- runtime memory 宿主
- capture / promote / recall / delete
- evidence loop
- 行为验证优先级

## 上下文

- 当前项目已经有最小 runtime 宿主，但 reuse 证据仍然很少
- 当前主线应是行为验证，不是继续扩 memory 复杂度
- 研究结果应直接影响当前项目的下一步修正

## 输入

- 当前项目的 runtime 结构和说明文档
- 官方文档 / 项目资料：
  - LLM
  - Aider
  - LangMem
  - Mem0 / OpenClaw

## 执行原则

- 少选对象，深看机制
- 优先官方资料
- 只提对当前项目阶段真的有用的结论
- 明确区分“值得借鉴”和“现在不该做”

## 实施步骤

1. 定义研究问题
2. 选定参考对象
3. 提炼每个对象最值得借鉴的一层
4. 总结共同模式和关键差异
5. 反推到当前项目的最小可行调整建议

## 输出要求

- 使用中文
- 明确结论导向
- 给出直接影响当前项目的决策建议

## 自检要求

- 这份研究是否帮助当前项目少走弯路
- 是否避免了把大项目机制直接套给当前项目
- 是否能直接导出当前项目的轻量修正动作
```
