## 1. 验证目标

## 4. 手工行为样例检查

本轮不启动新的真实客户端评测，但要做文档级行为模拟：

### 样例一：需求不完整的长链路任务

预期行为：

- 先进入需求澄清
- 达到高置信后做共识确认
- 确认后先写测试与验收方案
- 再写执行计划
- 执行后按预设验证回验

### 样例二：目标清楚的 Quick Path 任务

预期行为：

- 不过度追问
- 仍列出最小验收检查
- 验证计划和执行计划可以合并
- 完成后按检查项回验

### 样例三：非代码文档任务

预期行为：

- 使用口径一致性、证据支持、结构完整性、读者路径等人工验收方法
- 不因为没有自动化测试就跳过验证

## 5. 失败处理

任何必过验证项失败时：

1. 先定位失败文件或规则
2. 修正相关文档或脚本
3. 重新运行对应 `rg` 检查或完整校验脚本
4. 必过项全部通过后，才能进入最终反馈

如果某项无法通过，必须说明外部阻塞和影响范围，不能用自评分替代验证。
