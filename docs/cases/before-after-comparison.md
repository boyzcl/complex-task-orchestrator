# before / after 对照

## 资产类型

`curated comparison`

## 说明

这里的 `before` 是一个重建的直接执行基线，用来说明不使用协议时最容易出现的失真点。

它不是一份原始历史日志。

## 对照任务

研究型复杂任务：围绕“性行为相关的能力和欲望受哪些因素影响”做综述。

## before：直接执行基线

### 典型输入

```text
帮我全面写一篇关于性能力和性欲受哪些因素影响的文章。
```

### 高概率问题

- 容易把欲望、能力、满意度混为一谈
- 容易不先设证据等级，直接堆信息
- 容易滑向建议、疗法或经验性判断
- 容易把观察性关联写成确定因果

### 典型结果风险

- 内容看起来全面，但结构不稳
- 证据强弱混杂
- 输出边界不清

## after：使用 skill 协议

### 关键前置动作

- 先明确研究问题
- 先区分“欲望”和“能力”
- 先确认高等级证据优先
- 先定义不提供改善建议

### 结果变化

- 输出边界更清楚
- 证据层级更明确
- 结论与不确定性分得更清楚
- 任务过程中留下了需求共识、执行指令和自检总结

## 关键差异

| 维度 | 直接执行 | 使用 skill |
| --- | --- | --- |
| 任务对齐 | 容易混 | 先对齐再做 |
| 证据策略 | 容易后置 | 前置确认 |
| 文档留痕 | 通常没有 | 默认有最小工件 |
| 失真控制 | 靠临场判断 | 靠协议约束 |
| 自检 | 容易省略 | 默认执行 |

## 结论

这类 skill 的价值不在于“多问几句”，而在于把复杂任务里最容易导致返工的部分前置处理。
