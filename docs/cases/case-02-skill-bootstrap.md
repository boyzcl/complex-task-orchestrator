# 案例 2：skill 自举创建任务

## 资产类型

`curated case`

## 任务类型

workflow / skill 设计与产品化

## 原始任务

把原本散落在协作习惯和元生成指令中的复杂任务范式，整理成一个可安装、可调用、可复用的 skill。

## 为什么直接做容易跑偏

- 很容易直接开始写规则，却没有先定义“这个 skill 到底解决什么问题”
- 很容易把项目展示层和运行层混在一起
- 很容易把“我自己会用”误判成“别人也能复用”

## skill 是怎么介入的

这次任务本身就按这套协议执行：

- 先做需求共识
- 再生成一次性执行指令
- 再构建 skill 本体
- 最后自检并给出扣分原因

## 最终产物

- `skill/complex-task-orchestrator/`
- 需求共识文档
- 执行指令文档
- 自检与总结文档
- 本地安装副本

## 结果价值

- 让隐性的协作手感变成了显性协议
- 把“复杂任务起手式”做成了可安装 skill
- 把阶段感、自检、轻量/完整模式都显性化

## 学到的模式

- 创建 workflow 类 skill 时，最好先把自己也放进协议里跑一遍
- 运行时用户文档要优先跟随当前工作语言
- 阶段播报对复杂任务体验很关键

## 证据边界

该案例基于本地运行样本整理，不是原始运行日志直出。
