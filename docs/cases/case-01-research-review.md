# 案例 1：研究 / 综述任务

## 资产类型

`curated case`

## 任务类型

研究 / 综述 / 证据整合

## 原始任务

围绕“性行为相关的能力和欲望受哪些因素影响”完成一份中文循证综述。

## 为什么直接做容易跑偏

- “欲望”和“能力”很容易被混成一个问题
- 如果不先确定证据优先级，检索会快速失控
- 如果不先说明是否提供建议，输出边界会变得很乱

## skill 是怎么介入的

在正式检索前，先通过协议完成：

- 范围确认
- 输出边界确认
- 证据等级优先级确认
- 一次性执行指令生成

## 最终产物

- 需求共识文档
- 任务执行指令
- 中文综述正文
- 自检与总结文档

## 结果价值

- 最终结果把“欲望”和“能力”分开处理
- 明确标注了证据强弱与不确定性
- 输出边界更稳，没有滑向治疗建议

## 学到的模式

- 研究任务要先定义问题，再搜证据
- 证据优先级必须先讲清楚
- 不确定性必须被单独保留
