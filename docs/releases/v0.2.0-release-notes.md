# v0.2.0 Release Notes

日期：2026-04-17

## 版本定位

`v0.2.0` 代表这个项目已经从“有协议设计”推进到“有最小真实验证、最小 runtime reuse、最小治理规则”的阶段。

它不代表成熟产品，但代表：

- 协议层已经可安装、可执行、可说明
- Round 1 的最小行为验证已经完成
- runtime memory 已有真实 reuse 证据
- memory 治理开始被显式纳入设计，而不是继续隐含处理

## 本版本包含什么

### 1. 技能本体更完整

- `SKILL.md` 已包含 `Quick Path`
- 已包含 `Runtime Reuse Rules`
- 已包含 `Failure Handling`
- 已包含 `Success Signals`
- 已补入面向重复验证任务的脚手架意识

### 2. 公开证据层更完整

- 3 个公开案例
- 1 个 before / after 对照
- Round 1 的 6 个任务评分卡
- `v0.2.0` release gate 与 readiness assessment

### 3. runtime memory 进入最小闭环

- 有 capture
- 有 promoted pattern
- 有 promoted failure mode
- 有 3 条真实 reuse 记录
- manifest 已升级为带轻量治理字段的索引

### 4. 版本与验证面更清楚

- changelog 已成形
- validation matrix 已覆盖 release、Round 1、Round 2 framework 和 governance
- 新增 `Round 2` 客户端验证计划与脚手架

## 本版本明确不宣称什么

- 不宣称 Codex 与 Claude Code 的所有真实热路径都已充分验证
- 不宣称存在自动化 memory orchestration
- 不宣称 promoted 资产已经足够大或足够稳定
- 不宣称长期 recall 质量已经被证明

## 为什么现在进入 v0.2.0

因为当前项目已经满足自己定义的最低 gate：

- 6 个任务评分
- 至少 4 个 `Clearly helpful`
- 至少 1 个 `Quick Path` 真实通过案例
- 至少 2 条真实 reuse 记录
- 至少 1 条新 failure mode
- 至少 1 轮 skill 本体修正

## 下一阶段

`v0.2.x` 的主线应集中在两件事：

1. `Round 2` 客户端真实行为验证
2. 轻量 memory governance 的持续校准

等这两件事形成更强证据后，再考虑 `v0.3.0`。
