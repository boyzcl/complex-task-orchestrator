# 需求共识

## 当前任务

创建一个可复用的 Skill，名称为 `complex-task-orchestrator`，并且把这次创建过程本身当作一次完整的复杂任务流程来执行。

## 目标

构建一个流程型 Skill，让 Agent 能够：

- 通过一问一答式追问来澄清复杂需求
- 在执行前与用户达成明确共识
- 为当前任务生成一份一次性的 Markdown 执行指令
- 基于该指令推进任务执行
- 在完成后进行自检、修订，并输出最终总结

## 范围

- 以 [../元生成指令.md](../元生成指令.md) 作为核心输入之一
- 为这次流程保留关键项目文档
- 在 `skill/complex-task-orchestrator/` 下创建可复用 Skill
- 将该 Skill 安装到本地 Codex Skill 环境中
- 验证 Skill 的结构与本地可用性

## 已确认流程

1. 从“最强大脑”的视角出发，先定义问题，再组织方案。
2. 通过一次一个问题的方式逐步澄清需求。
3. 当理解置信度足够高时，停止追问并复述需求。
4. 复述后等待用户明确确认。
5. 在正式执行前，先写出当前任务的一次性 Markdown 执行指令文档。
6. 基于该文档执行任务。
7. 执行后进行自检、必要时修订，并完成总结。

## 行为规则

- 默认使用偏严格的流程，除非任务已经足够明确，可以合理压缩。
- 该流程用于复杂、开放式、多阶段或需求不完整的任务。
- 不要把简单、低风险任务强行套进重流程。
- 如果新的歧义会改变范围、方向或验收标准，必须暂停执行并回到澄清阶段。
- 如果歧义只影响局部细节、不影响主路径，可以做合理假设继续推进，并在结果中说明。
